# 🚀 Global Shofar Academy: 배포 및 서비스 전략 제안서

> **Goal**: 3개 국어(한/영/대)로 완성된 Shofar 커리큘럼을 전 세계 사용자가 가장 효과적으로 학습할 수 있는 플랫폼 구축.

---

## 1. 옵션 분석 (Options Analysis)

이미 우리는 **완벽한 데이터(Markdown Files)**를 가지고 있습니다. 이를 어떻게 "서빙"하느냐의 문제입니다.

### Option A: 🌐 지식 웹사이트 (Knowledge Site) - **[강력 추천]**
Obsidian의 노트를 그대로 웹사이트로 변환하여 퍼블리싱하는 방식입니다.
*   **장점**:
    *   **즉시 구축**: 현재 파일 구조(링크, 태그)를 100% 그대로 살릴 수 있습니다.
    *   **접근성**: 구글 검색(SEO)에 노출되어 전 세계 유입이 가능합니다.
    *   **비용**: GitHub Pages 등을 이용하면 호스팅 비용이 **0원**입니다.
*   **추천 도구**: **Quartz** 또는 **Docusaurus**.
*   **적합성**: ★★★★★ (현재 데이터 형태에 최적)

### Option B: 🤖 RAG AI 챗봇 (AI Tutor)
사용자가 질문하면 커리큘럼 내용을 바탕으로 답변해 주는 AI입니다.
*   **장점**:
    *   **맞춤형 교육**: "초보자가 앙부슈르 잡는 법 알려줘"라고 물으면 즉시 답변.
    *   **상호작용**: 24시간 질문 답변 가능.
*   **단점**:
    *   **전체 맥락 부족**: 챗봇은 파편화된 정보만 줍니다. 커리큘럼 전체("A부터 Z까지")를 공부하기엔 부족합니다.
    *   **비용**: LLM API 호출 비용 발생.
*   **적합성**: ★★★☆☆ (웹사이트의 보조 도구로 적합)

### Option C: 🎓 LMS (온라인 강의 시스템)
Moodle, Teachable, Udemy 같은 플랫폼.
*   **장점**:
    *   **수료증/관리**: 누가 어디까지 들었는지, 시험 점수는 몇 점인지 관리 가능.
    *   **비즈니스**: 유료 강의 판매 용이.
*   **단점**:
    *   **구축 난이도**: 데이터를 플랫폼에 일일이 다시 입력해야 함.
    *   **비용**: 플랫폼 이용료 비쌈.
*   **적합성**: ★★☆☆☆ (120명 인증이 급할 때만 고려)

---

## 2. 🏆 최적의 솔루션: "하이브리드 전략"

가장 현실적이고 강력한 방법은 **Option A(웹사이트)를 메인으로 하고, Option B(AI)를 붙이는 것**입니다.

### 1단계: Quartz로 "Global Shofar Wiki" 오픈
*   Obsidian Vault를 그대로 웹사이트로 변환해 주는 오픈소스 도구 **[Quartz](https://quartz.jzhao.xyz/)**를 사용합니다.
*   **특징**:
    *   `[[WikiLink]]`가 그대로 웹 링크로 작동.
    *   3개 국어 폴더 구조 완벽 지원.
    *   매우 아름답고 빠름 (이미 전 세계 지식 공유자들의 표준).

### 2단계: 사이트 내 "Rabbi Shofar" 챗봇 탑재
*   구축된 웹사이트 우측 하단에 말풍선 버튼을 답니다.
*   **기능**:
    *   "3주차 연습 스케줄 짜줘"
    *   "Teruah 소리가 안 나는데 팁 있어?"
    *   이 질문들에 대해 우리 `Obsidian Vault` 문서를 근거(Reference)로 답변.

---

## 3. 실행 로드맵 (Execution Plan)

지금 바로 실행할 수 있는 구체적인 단계입니다.

1.  **웹 퍼블리싱 (Publishing)**:
    *   `npx quartz create` 명령어로 프로젝트 생성.
    *   현재 Obsidian Vault 파일을 Quartz 폴더로 복사.
    *   GitHub Pages로 배포 (무료).
    *   도메인 연결 (예: `shofar-academy.com`).

2.  **RAG 구축 (AI Integration)**:
    *   오픈소스 챗봇 UI(예: Chatbot UI) 사용.
    *   우리의 `.md` 파일들을 벡터 DB에 임베딩.
    *   (선택) 더 쉬운 방법: **Dify**나 **GPTs**에 파일들을 업로드하고 링크 공유.

1.  **미국 사용자 (John)**: 구글에 "Shofar technique" 검색 -> 우리 사이트 접속.
2.  **학습**: `English_Curriculum/Module 3`를 읽으며 영상 시청.
3.  **질문**: 이해 안 되는 부분은 우측 하단 챗봇에게 "How to breathe?" 질문.

5.  **대만 사용자 (王先生)**: `[🇹🇼]` 버튼 클릭하여 번체 본문 학습.
